# 🎨 CSS Style Guide

Цей документ встановлює правила написання чистого CSS-коду для фронтенд-команд.

---

## 📁 Структура та організація

- Файли стилів мають бути розбиті за компонентами, утилітами, темами.
- Іменування файлів: `component-name.css`, `utilities.css`, `layout.css`, `theme.css`
- Один компонент — один CSS-файл, якщо не використовується scoped styles або CSS-in-JS.

---

## 🧠 Іменування класів

- Використовуй lowercase-with-hyphens: `.btn-primary`, `.form-group`
- Рекомендовано БЕМ-нотацію:
  - Блок: `.card`
  - Елемент: `.card__title`
  - Модифікатор: `.card--highlighted`
- Уникай скорочень, якщо вони не загальновживані (`.btn` — ок, `.crd` — ні)
- Уникай загальних назв: `.red`, `.big`, `.box` — надавай контекст

---

## 🎯 Специфікація та селектори

- Уникай надмірної специфічності: не використовуй вкладеність > 3 рівнів
- Не використовуй ID-селектори (`#header`) — лише класи
- Уникай універсальних селекторів (`*`) у продуктивному коді
- Використовуй `:where()` або `:is()` для зниження специфічності, якщо потрібно

---

## 🎨 Стилі та форматування

- Відступи: 2 пробіли
- Порядок властивостей:
  1. Layout: `display`, `position`, `top`, `left`, `z-index`
  2. Box model: `width`, `height`, `margin`, `padding`, `border`
  3. Typography: `font-family`, `font-size`, `line-height`, `text-align`, `color`
  4. Visual: `background`, `box-shadow`, `opacity`
  5. Interaction: `transition`, `animation`, `cursor`
- Групуй логічно: розділяй блоки коментарями

---

## 📐 Адаптивність

- Використовуй медіа-запити з мобільного вгору (mobile-first)
- Уникай фіксованих розмірів (px) — надавай перевагу em, rem, %, vw, vh
- Використовуй clamp() для адаптивних шрифтів і відступів

## 🌐 Темізація

- Використовуй CSS-перемінні (--primary-color) для кольорів, шрифтів, spacing
- Темна/світла тема — через root-перемінні:

```css
:root {
  --background-color: #fff;
  --text-color: #000;
}

[data-theme="dark"] {
  --background-color: #000;
  --text-color: #fff;
}
```

## ♿ Accessibility

- Використовуй семантичні елементи (<button>, <nav>, <main>)
- Забезпечуй видимий фокус (outline, :focus-visible)
- Уникай display: none для прихованих елементів — краще visibility: hidden або aria-hidden
- Перевіряй контрастність кольорів (WCAG AA/AAA)

## ✅ Коментарі

- Коментарі мають пояснювати "чому", а не "що"
- Формат: /_ Центрує контент по вертикалі _/
- Уникай закоментованого коду — видаляй, якщо не використовується

## 🚫 !important

Уникай використання `!important`, якщо можливо — використовуй специфічність селекторів!

## 🧪 Перевірка та лінтинг

- Використовуй Stylelint з базовими правилами
- Автоматичне форматування через Prettier
- CI має включати перевірку стилів

## 📌 Відповідальність

- Кожен розробник дотримується цього Style Guide
- Техлід контролює відповідність при Code Review
- Зміни до Style Guide — через RFC або командне погодження
