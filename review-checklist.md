# ✅ Pull Request Review Checklist

Цей чекліст призначений забезпечити якість, стабільність і узгодженість коду при перегляді Pull Request'ів у фронтенд-проєктах.

---

## 📦 Загальні критерії

- [x] PR має зрозумілу назву та опис змін
- [x] Пов’язані задачі/issue згадані (наприклад, `Closes #123`)
- [x] Зміни логічно згруповані, без зайвого коду
- [x] Всі коментарі та тимчасовий код (`console.log`, `debugger`) видалені



## 🧪 Тестування

- [x] Додані або оновлені **unit-тести** для нового/зміненого функціоналу
- [x] Тести проходять локально (`npm test`, `vitest`)
- [x] Покриття коду не зменшилось нижче допустимого порогу
- [x] Всі мок-дані та фікстури коректно організовані



## 🧱 Код та архітектура

- [x] Дотримано **стилю коду** (ESLint, Prettier або oxlint/oxfmt)
- [x] Іменування змінних, функцій, компонентів — зрозуміле та відповідає [прийнятим правилам](naming.md)
- [x] Відсутні дублікати логіки або неефективні конструкції
- [x] Код розбитий на **чіткі, ізольовані блоки**
- [x] Всі утиліти, хуки, компоненти — в правильних директоріях



## 🧩 Компоненти та UI

- [x] Компоненти мають коректні props, типи, default значення
- [x] Всі стилі адаптовані для світлої/темної теми (якщо застосовується)
- [x] Відсутні візуальні артефакти або порушення дизайну
- [x] Всі інтеракції протестовані (клік, фокус, введення)



## 📚 Документація

- [x] Оновлено README / документацію, якщо зміни публічні
- [x] Коментарі до складної логіки — зрозумілі та лаконічні
- [x] Нові утиліти, функції, компоненти — мають JSDoc/опис



## 🔐 Безпека та стабільність

- [x] Всі зовнішні API-запити обробляють помилки
- [x] Відсутні потенційні XSS/CSRF ризики
- [x] Зміни не порушують існуючу логіку або інші модулі
- [x] Перевірено поведінку в різних браузерах/екранах (якщо релевантно)



## 🚀 CI/CD та деплой

- [x] CI проходить без помилок
- [x] Всі build-скрипти працюють (`npm run build`)
- [x] Зміни не порушують деплой або інфраструктуру



## 🧊 Додатково

- [x] Всі обговорення в PR вирішені
- [x] PR готовий до злиття без додаткових правок
