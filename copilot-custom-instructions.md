# Copilot Custom Instructions

## Visual Studio Code

VS Code підтримує три типи користувацьких інструкцій репозиторію. Щоб дізнатися більше про те, які функції GitHub Copilot підтримують ці типи інструкцій:

- **Налаштовувані інструкції для всього репозиторію**, які застосовуються до всіх запитів, зроблених у контексті репозиторію. Вони вказані у файлі `copilot-instructions.md` у каталозі `.github` репозиторію.
- **Спеціальні інструкції для кожного шляху**, що застосовуються до запитів, зроблених у контексті файлів, що відповідають заданому шляху. Вони вказані в одному або кількох файлах `NAME.instructions.md` у каталозі `.github/instructions` у репозиторії. Якщо вказаний вами шлях збігається з файлом, над яким працює Copilot, і також існує файл користувацьких інструкцій для всього репозиторію, тоді використовуються інструкції з обох файлів.
- **Інструкції агента** використовуються агентами штучного інтелекту. Ви можете створити один або декілька файлів `AGENTS.md`, які зберігатимуться будь-де в репозиторії. Під час роботи Copilot найближчий файл `AGENTS.md` у дереві каталогів матиме пріоритет.

### Створення користувацьких інструкцій для всього репозиторію

1. У кореневому каталозі вашого репозиторію створіть файл з назвою `.github/copilot-instructions.md`. Створіть каталог `.github`, якщо його ще не існує.
2. Додайте інструкції природною мовою до файлу у форматі `Markdown`. Пробіли між інструкціями ігноруються, тому інструкції можна писати як окремий абзац, кожну з нового рядка, або розділяти порожніми рядками для зручності читання.

### Створення спеціальних інструкцій для кожного шляху

1. Створіть каталог `.github/instructions`, якщо його ще не існує.
2. Створіть один або декілька файлів `NAME.instructions.md`, де NAME вказує на призначення інструкцій. Ім'я файлу має закінчуватися на `.instructions.md`.
3. На початку файлу створіть блок `frontmatter`, що містить ключове слово `applyTo`. Використовуйте синтаксис `glob`, щоб вказати, до яких файлів або каталогів застосовуються інструкції. Ви можете вказати кілька шаблонів, розділяючи їх комами.

Приклад:

```
---
applyTo: "src/**/*.ts,**/*.vue,**/*.css"
---
```

Щоб застосувати інструкції до всіх файлів, використовуйте `applyTo: "**"`, `applyTo: "*"` або `applyTo: "**/*"`.

4. За потреби, щоб запобігти використанню файлу агентом кодування Copilot або рецензентом коду Copilot, додайте ключове слово `excludeAgent` до блоку `frontmatter`. Використовуйте або `code-review`, або `coding-agent`.

Наприклад, наступний файл буде прочитаний лише агентом кодування Copilot:

```
---
applyTo: "**"
excludeAgent: "code-review"
---
```

Якщо ключове слово `excludeAgent` не включено до `frontmatter`, ваші інструкції використовуватимуть як перевірка коду Copilot, так і агент кодування Copilot.

5. Додайте власні інструкції природною мовою, використовуючи формат `Markdown`. Пробіли між інструкціями ігноруються, тому інструкції можна писати як окремий абзац, кожен на новому рядку, або розділяти порожніми рядками для зручності читання.

### Що таке Frontmatter

**Front Matter** - це блок метаданих, який розташовується на початку Markdown файлу. Він використовується для опису та зберігання інформації про документ (метадані документа), такий як заголовок, опис, автор, дата створення, дата публікації, категорія, теги та інші параметри. Front Matter є розширенням стандартного синтаксису Markdown і дозволяє створювати складніші та структурованіші документи.

Front Matter починається з `---` і закінчується `---`. Обмежувачі мають находитись на окремих рядках.

```
---
# Тут дані front matter'a
---
```

Мета використання Front Matter полягає в тому, щоб полегшити керування контентом у файлових системах керування контентом та автоматизувати процес створення статичних веб-сайтів генераторами статичних веб-сайтів. Він дозволяє швидко та легко налаштовувати метадані для кожного документа та генерувати відповідний HTML-код, об'єднувати на основі метаданих окремі документи у колекції.

Перевага Front Matter перед іншими варіантами зберігання метаданих документа полягає у спільному зберіганні метаданих та змісту документа в одному текстовому файлі. При цьому сам текстовий формат Markdown файлу має на увазі простий спосіб внесення змін до документа за допомогою будь-якого доступного текстового редактора.

Це стосується і блоку Front Matter, дані в якому зберігаються у зрозумілому для людини вигляді. Використовувані формати даних Front Matter залежать від вибраного генератора статичних сайтів або системи управління контентом. Деякі можуть використовувати формати даних YAML, JSON або TOML. Найбільш поширений формат даних YAML, який, як правило, використовується не тільки для зберігання метаданих документів у Front Matter, але і для зберігання налаштувань системи в конфігураційних файлах, що дозволяє досягти одноманітності.

Загалом, Front Matter є корисним інструментом для керування контентом та автоматизації процесу створення статичних веб-сайтів. Він дозволяє швидко і легко налаштовувати метадані для кожного документа, забезпечує переносимість даних між різними генераторами статичних веб-сайтів і системами керування контентом, без залежності від мов програмування та технологій, що використовуються для їх розробки, спрощує процес генерації HTML-коду.
